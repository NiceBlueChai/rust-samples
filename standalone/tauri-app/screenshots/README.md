# 截图目录说明

## 📁 目录结构
```
screenshots/
├── login.png           # 登录界面截图
├── user-info.png       # 用户信息页面
├── product-list.png    # 产品列表页面
├── bug-list.png        # Bug列表页面
├── column-settings.png # 列设置对话框
├── bug-detail.png      # Bug详情页面（可选）
└── about.png           # 关于页面（可选）
```

## 📝 截图规范

### 尺寸建议
- **主界面截图**: 1200x800px 或更大
- **对话框截图**: 600x400px 或适合内容的尺寸
- **保持统一**: 所有截图使用相同的缩放比例

### 命名规范
- 使用英文小写字母
- 单词间用连字符 `-` 分隔
- 文件格式建议使用 PNG（清晰度好，支持透明背景）

### 内容要求
- 确保截图内容清晰可读
- 避免包含敏感信息（如真实的服务器地址、账号密码）
- 可以使用示例数据或模糊处理敏感内容

### README 中的引用
```markdown
![描述文字](./screenshots/文件名.png)
```

## 🎯 推荐的截图内容

1. **login.png**: 登录界面，展示表单验证和安全提示
2. **user-info.png**: 用户信息页面，展示用户详细信息
3. **product-list.png**: 产品列表，展示产品数据表格
4. **bug-list.png**: Bug列表，展示自定义列功能
5. **column-settings.png**: 列设置对话框，展示自定义功能
6. **bug-detail.png**: Bug详情弹窗（如果有的话）
7. **about.png**: 关于页面

## 📷 截图技巧

- 使用应用的实际运行界面
- 可以填充一些示例数据让界面更丰富
- 确保界面元素完整显示
- 可以使用截图工具的标注功能突出重点功能
